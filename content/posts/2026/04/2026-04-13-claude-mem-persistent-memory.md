Claude Code に永続的な記憶機能を追加する OSS ツール **claude-mem** が、公開からわずか 48 時間で **46,000 スター**を獲得し、大きな話題となっています。「トークン消費を 95% 削減」「コンテキスト上限に達しない」「前回の続きから完全に再開」という特徴が開発者の注目を集めました。

## claude-mem とは

[claude-mem](https://github.com/thedotmack/claude-mem) は、Claude Code のセッションをまたいで記憶を保持するための MCP（Model Context Protocol）ベースのプラグインです。通常の Claude Code はセッションを閉じると会話の文脈が失われますが、claude-mem を導入することで過去のセッションで得た知識・コンテキストを次回以降に引き継げます。

## 主な機能

### 自動的な記憶の収集と圧縮

Claude Code での操作をすべて自動キャプチャし、AI を使って記憶を圧縮・要約します。ローカルの SQLite データベースに保存されるため、外部サービスへの依存がありません。

### 3層の検索フロー

関連する記憶を効率よく取り出すために、以下の 3 段階の検索を組み合わせています:

1. **キーワード検索** — 単純なテキストマッチ
2. **ベクトル埋め込み検索** — Chroma を使った意味的な類似検索
3. **ナレッジグラフ** — 関係性を考慮した検索

この仕組みにより、無関係な記憶を注入せず、必要な情報だけを選んでコンテキストに追加します。

### トークン消費を 95% 削減

MCP ツールの `search` → `timeline` → `get_observations` という 3 段階のワークフローで、先に絞り込みを行ってから詳細を取得します。これによりセッションあたりのトークン消費が大幅に削減され、公称で約 95% の削減を実現しています。

## インストール方法

1 コマンドでインストールできます:

```bash
npx claude-mem install
```

完全無料・オープンソースで、特別な API キーや外部サービスの登録は不要です。

## なぜ話題になったのか

Claude Code は高機能な AI コーディングアシスタントですが、セッションをまたぐ記憶の欠如は大きな課題でした。毎回プロジェクトの背景やコーディング規約を説明し直す手間があり、コンテキストウィンドウを圧迫するという問題もありました。

claude-mem はこの痛点をシンプルな 1 コマンドインストールで解消します。「永遠にコンテキスト上限に達しない」「前回中断した場所から完全に再開できる」というメリットが、多くの Claude Code ユーザーに刺さったと考えられます。
