Claude Code を使っていて「なんかコストかかるな…」と思ったことはないでしょうか。実はデフォルト設定のまま使い続けると、推定 2 億 6,400 万トークンもの無駄が発生することが判明したという事例が話題になっています。

## デフォルト設定の罠: ENABLE_TOOL_SEARCH

問題の根本は `settings.json` の `ENABLE_TOOL_SEARCH` 設定です。

デフォルトで有効になっているこの設定が、コンテキストを大量に消費していた原因でした。`settings.json` に以下を追加するだけで、コンテキストが **45K トークンから 20K トークンへ即座に低下** したという報告があります。

```json
{
  "ENABLE_TOOL_SEARCH": "true"
}
```

ただし、この設定を明示的に記述することで、Claude Code 側の挙動が変わり、不要なツール検索をスキップするようになります。長期セッションで大量にトークンを消費している場合、まず確認すべき設定です。

## キャッシュの有効期限切れも要注意

もう 1 つの落とし穴が **プロンプトキャッシュの有効期限**です。

Claude のプロンプトキャッシュは **5 分**で期限切れになります。5 分休憩しただけで、会話全体が再処理され、コストが **10 倍以上**に跳ね上がることがあります。

つまり:

- 長時間セッションの途中で離席する
- ちょっと休憩してから作業再開する
- 別の作業をしてから Claude Code に戻ってくる

といった行動がすべて、想定外のコスト増につながる可能性があります。「休憩明けの最初のメッセージが一番高い」というのは、このキャッシュ再処理が原因です。

## 対策まとめ

| 問題 | 原因 | 対処法 |
|------|------|--------|
| コンテキストが肥大化する | `ENABLE_TOOL_SEARCH` のデフォルト挙動 | `settings.json` に明示的に設定を追加 |
| 休憩後にコストが急増する | キャッシュの 5 分 TTL | 長期セッションでは連続作業を心がける |

## Claude Code の設定を見直すタイミング

Claude Code を使い始めたばかりの方、または長期間デフォルト設定のまま使い続けている方は、一度 `settings.json` を見直してみることをおすすめします。

特に:

- **長時間セッション** を多用している
- **大規模なコードベース** で作業している
- **API コストが予想より高い** と感じている

といった場合には、設定の最適化で大幅なコスト削減が期待できます。

## まとめ

Claude Code は強力なツールですが、デフォルト設定のまま使い続けると意図せず大量のトークンを消費してしまうことがあります。`ENABLE_TOOL_SEARCH` の明示設定と、キャッシュ有効期限を意識した作業フローの見直しで、コストを大幅に抑えられます。

設定ファイルのチューニングは地味ですが、長い目で見ると大きな効果をもたらします。まだ設定を確認していない方は、ぜひ一度チェックしてみてください。
